2024 重庆大学软件学院 计算机组成与结构 笔记

考试比较基础

  1. 流水线级数 越多越好？
    NO！ 流水线对 单条指令的执行时间 基本没有影响， 甚至会 增加执行时间。 因为流水线有 额外开销 —- 锁存器时延和时钟偏移时延。

    额外开销限制了流水线深度的增加。当流水线时钟周期减少到和额外开销时间相等时，流水线就起不到提高性能的作用了。

    • 流水线没有减少单条指令的执行时间，甚至增加了少许—-由于额外开销
    • 流水线主要提高了 吞吐率

  3. 流水线优化技术

    • 结构冲突—-增加功能单元，功能单元流水
    • 数据冲突—-停顿；forwarding(前向技术)；编译器将指令隔开
    • 控制冲突—-flushing(流水线清空)；non-taken(预测分支失败)；taken(预测分支成功)；分支延迟(延迟槽)—-从前，失败处，目标处
    • 动态分支预测—-分支历史表(BHT);相关分支预测(m,n);分支目标缓冲(BTB)
    • 循环展开(unrolling)
      • 重复循环体多次
      • 合并不必要的指令
      • 换名消除相关
      • 调度调整指令顺序
    • 动态调度—-记分牌(Scoreboard)；Tomasulo算法

  4. 局部性原理

    • 时间局部性：在一个时间点被访问的资源，在将来被再次访问的可能性更大
    • 空间局部性：在一个时间点被访问的资源，邻近它的资源在将来有更大的可能性被访问。
    • 90/10法则：一个程序在执行时，90%的执行时间在运行10%的代码
    • Cache CPU访问Cache使用的是 时间局部性， 而Main Memory将一个Block置换到Cache中利用的是 空间局部性

  5. 三种指令集结构

  6. 相关(dependence)和冲突(hazard)

    • dependence—-真实相关(RAW); 名相关—anti相关(WAR); 输出相关(WAW)
    • hazard—-结构冲突；数据冲突；控制冲突
    • 相关不一定导致冲突
    • 是否有冲突根据代码具体分析；但是相关一定要列完整
    • 真实相关具有 传递性

  7. 写回 与 写直达——按写分配 与 不按写分配

    • 写回
      • 执行”写操作”时，只需将数据写回Cache中对应的块，不用写回到Main Memory中。只有当Cache中的块被置换时，才需要根据”dirty bit”决定是否将块写回Main Memory中
      • 优点：
        1. 写操作能够以Cache的速度进行
        2. 对同一单元的多次写操作，最后只需要一次写回到下一级存储器中
        3. 对没有进行修改的块，置换时不需要写回Main Memory中
        4. 写回减少了Main Memory的带宽压力和功耗
    • 写直达
      • 执行”写操作”时，需要将数据写回到Cache和Main Memory中对应的块。
      • 优点：
        1. 常常采用”写缓冲器(write buffer)”来减少”写停顿”
        2. 实现简单
        3. 使得Main Memory中的数据保持最新，和Cache中一致
    • 按写分配——用于写回：写不命中时，需要将写操作对应的块从Main Memory中置换到Cache中，再进行写操作
    • 不按写分配—-用于写直达：写不命中时，直接写入Main Memory中的对应块

  9. 记分牌算法 和 Tomasulo算法

    • 相同点： 顺序发射，乱序执行，乱序完成
    • 不同点：
      • tomasulo通过寄存器换名过程可以消除WAR和WAW冲突； scoreboard 能检测WAR和WAW冲突，一旦检测到冲突，通过插入停顿周期来解决这一冲突——记分牌不能消除WAR和WAW冲突
      • 检测冲突和控制指令执行方式不同
        • Tomasulo方法检测冲突和控制指令执行两方面功能是通过分布在每一功能单元的保留站来进行的，因此Tomasulo方法是一种分布式方法。
        • 记分牌方法的上述功能是通过统一的记分牌来实现的，因此记分牌方法 是一种集中式方法
      • 写结果的方法不同
        • Tomasulo方法通过CDB直接将功能单元输出的结果送往需要该结果的所有保留站，而不必经过寄存器这一中间环节。
        • 记分牌方法是将结果写入FP寄存器, 因而可能造成等待这一结果的指令都出现停顿现象，之后，所有相关指令的功能单元在读FP 寄存器时又可能 出现竞争现象。
      • Tomasulo方法中无检查WAW和WAR竞争功能，因为在指令发射过程中，由 issue logic结合保留站完成了register operands的改名过程，即消除了这两冲突
      • 记分牌存在结构相关和WAW相关时无法发射后续指令，导致整个流水线空闲；没有转发操作；

Amdahl 定律

如果组件 $x$ 被提高了 $S_x$, 同时组件 $x$ 占原来执行时间的 $F_x$，那么加速比 $Speedup$ 为：

$ \text{Speedup} = \frac{\text{OldExecutionTime}}{\text{NewExecutionTime}} = \frac{\text{NewPerformance}}{\text{OldPerformance}} = \frac{1}{(1-F_x)+\frac{F_x}{S_x} }$

  • $S_x \rightarrow \infty, speedup\rightarrow 1/(1-F_x)$
    • $S_x$ 越趋近于无穷，$F_x/S_x$ 越趋近于0， speedup增加的速度越慢
  • $F_x\rightarrow 1, speedup \rightarrow S_x$

Example: Amdahl

Chapter 4: Pipelining

流水线概念：将指令的处理过程分解为若干可区分的段(stage)，确保有足够的硬件资源使得在每个段能够处理不同的指令。

流水线注意点

  • 流水线无助于单任务的时延，只是提高了整体的吞吐率
    • 甚至会增加单指令的执行时间，由于流水线额外开销
  • 时间长的段将成为流水线的瓶颈。
    • 流水线的时钟周期 = 最长段时间
  • 潜在加速比 = 流水线深度
  • 不平衡的段会影响加速比
  • “填充” 和 “排空” 时间会影响流水线加速比

流水线的MIPS实现

  • register 前半周期写，后半周期读
  • Memory 分为数据 和 指令

MIPS特征

  • ISA编码：所有指令的长度相同
  • 存储器访问只通过 load/store 指令

流水线CCT计算

  • 告诉了每个stage的时间，那么最长的stage时间加上overload就是 流水线CCT
  • 只告诉了original时钟周期， 和overload，那么 两者相加就是 CCT

Chapter 6: Memory Hierarchy Design

为什么内存层次设计是必要的?

内存层次结构设计在计算机架构中至关重要，原因有几个，主要与性能、成本和能效有关。以下是详细解释：

  • 性能改进：

    • 速度差异：CPU和主存储器之间存在显著的速度差异。现代CPU可以在纳秒内执行指令，而从主存储器（DRAM）访问数据需要更长的时间，如果必须等待数据，会导致CPU空闲时间。内存层次结构，更快的内存类型更接近CPU，有助于减轻这种延迟。
    • 缓存利用：通过使用多级缓存（L1、L2、L3），系统可以利用局部性原则（时间和空间）将经常访问的数据保存在更快、更小的缓存中，从而减少平均访问时间。

  • 成本效率：

    • 性价比权衡：高速缓存中使用的SRAM等高速内存价格昂贵，不适用于大规模存储。用于主存储器的DRAM速度较慢，但更便宜。磁盘存储（HDD或SSD）甚至更便宜，用于海量、访问频率较低的数据。分层设计通过在不同级别使用不同类型的内存来平衡成本和性能。

  • 电源效率：

    • 能耗：与从主存或磁盘存储访问数据相比，从内存层次结构中的更高级别（如缓存）访问数据消耗的功耗更低。内存层次结构通过最大限度地减少对速度较慢、耗电的内存组件的访问来降低系统的整体能耗。

  • 局部性原则：

    • 时间局部性：最近访问的数据很可能很快再次访问的原则。通过将最近的数据保存在最快、最小的缓存（L1）中来利用此原则。
    • 空间局部性：位于最近访问的数据附近的数据可能很快就会被访问的原则。较大的缓存（L2、L3）和主存储器通过加载连续的数据块来利用这一点。

映射规则

直接映射

主存储器中的某块可以存放到Cache中的唯一位置

  • 查找简单快速，适合大型Cache
  • 给定一个Block唯一的固定位置，容易与其他块冲突，导致Cache不命中率较高

全相联映射

主存储器中的某块可以存放到Cache中的任意位置

  • 完全灵活随机的设置块在Cache中的位置
  • 查找代价较高，需要比较每个Block的Tag—-适合小型Cache
  • 最小的不命中率

多路组相联映射

主存储器中的某块可以存放到Cache中的唯一set中的任意位置

相联程度增加 -> #Block in Set增加 -> #Set减小 -> 查找时间增加 -> 不命中率减少

写策略

写回 与 写直达——按写分配 与 不按写分配

  • 写回
    • 执行”写操作”时，只需将数据写回Cache中对应的块，不用写回到Main Memory中。只有当Cache中的块被置换时，才需要根据”dirty bit”决定是否将块写回Main Memory中
    • 优点：
      1. 写操作能够以Cache的速度进行
      2. 对同一单元的多次写操作，最后只需要一次写回到下一级存储器中
      3. 对没有进行修改的块，置换时不需要写回Main Memory中
      4. 写回减少了Main Memory的带宽压力和功耗
  • 写直达
    • 执行”写操作”时，需要将数据写回到Cache和Main Memory中对应的块。
    • 优点：
      1. 常常采用”写缓冲器(write buffer)”来减少”写停顿”
      2. 实现简单
      3. 使得Main Memory中的数据保持最新，和Cache中一致
  • 按写分配——用于写回：写不命中时，需要将写操作对应的块从Main Memory中置换到Cache中，再进行写操作
  • 不按写分配—-用于写直达：写不命中时，直接写入Main Memory中的对应块
