第一章 Intro

重点：CS结构、OS组成、OS历史、UNIX例子、实时类别

  • 计算机系统的结构：硬件->操作系统->系统和应用程序（编译器，汇编）->用户
  • 操作系统是什么？
    • 控制程序，资源管理器，扩展机器
  • 操作系统组成(component)
    • CPU管理器、内存管理器、文件系统、设备管理器
  • 操作系统历史
    • 批处理系统、多道程序系统、分时操作系统、OS for 桌面机 和 嵌入式系统
    • 注意区别前3个
      • 批处理：一个时间点只能运行一个 —- IBM 1401
      • 多道程序：多个job在 内存 中，CPU复用
      • 分时：job存储在 内存和磁盘，只有内存中的job才能上CPU
  • 桌面操作系统
    • MS-DOS， Windows NT/XP
    • UNIX —- BSD UNIX、Solaris、Mac OS
    • Linux
  • 嵌入式操作系统
    • 关键词： 控制设备(control device) 有限资源
    • 可能有 时间要求，即实时性(Real time)
      • 硬实时: 在特定的时间点动作必须出现
      • 软实时： 偶尔错过时限要求可以接受

第二章 计算机系统结构

  • 整体结构

    • CPU和总线相连，其他如存储器、设备等 通过各自的控制器与总线相连
    • CPU和设备控制器 并发执行，竞争cycles。
    • 通过内存控制器实现 并发内存访问

  • Bootstrap(引导)

    • 将Kernel代码从 持久存储 加载到 内存中，并将控制转到OS入口。—— Bootstrap
    • 执行Bootstrap的程序叫做 boot-loader(引导器)

  • 中断(Interrupt)

    • 关键词： 中断驱动、IRQ(Interrupt Request)、IVT、IV(ISR地址）、ISR
    • 中断响应牵涉 —- 硬件执行（IP，SP，FLAGS) 和 ISR(汇编和C)
    • 中断出现 -> 硬件执行 -> 通过IRQ查询IVT找到对应的IV，跳转到ISR，执行ISR

  • 异常(Exception)

    • 异常—同步；中断—异步，外围设备触发
    • 异常：指令执行过程中检测到错误。 指令错误 除零错误，内存访问地址非法
    • 别称： SGI（Software generate Interrupt）和 同步中断

  • 如何访问外围设备

    • CPU通过 设备控制器 访问设备
    • 设备控制器 有两种寄存器： 命令寄存器 和 数据寄存器
    • 访问方式
      • IO 端口 IN/OUT —- IO指令，都是特权指令
      • 内存映射 IO —- 用访问内存的方式（即内存地址）访问设备寄存器。
        • 可寻址地址空间必须预留一部分作为 Memory map
        • 使用相同的总线来寻址内存和 I/O 设备
      • 同时使用： 命令传输用IO port； 数据传输用 memory mapped IO

  • 硬件保护

    • 方式： 二态模式 特权指令 CPU保护 存储器保护
    • 二态模式： 用户态 和 内核态 模式位(mode bit) 1=user
    • 特权指令
      • 所有的IO指令都是特权指令
      • 特权指令只能在 内核态 使用
      • 特权指令在用户态使用 触发 异常
    • CPU保护
      • Timer —- 定时器中断 —- 计数寄存器
    • 存储器保护 —- 确保内存访问地址合法，确定合法物理内存地址范围
      • base，limit register
      • logical address >= base -> logical address < base + limit

  • 系统调用(system call) —- OS与用户程序之间的接口

    • 系统调用是在 内核态
    • 用户程序只能通过系统调用使用OS提供的服务
    • 系统调用过程
      • 参数准备 — Wrapper — system call number in register — trap into OS — index system call SR in SC table — 执行服务例程 — 返回用户程序

  • 陷入OS

    • 异常(int 21h) 或者 特殊指令(SYSENTER/SYSEXIT)

  • 系统调用与库函数

    • 系统调用陷入OS，运行更慢； 库函数不陷入OS
    • 用户程序可以直接系统调用，也可以通过库函数进行系统调用

第七章 进程同步

  • 竞争条件：多个进程同时并发访问和操作同一数据的情况，执行结果取决于访问发生的特定顺序

  • 竞争条件避免

    • 禁止同时 访问和操作 共享数据
    • 访问必须 序列化

  • 临界区： 访问共享资源的代码段

    • 进入区，临界区，退出区，剩余区

  • 解决竞争条件必须满足

    • 互斥
    • 前进性(progress) — 空闲让进
    • 有限等待
    • 速度 — 对 CPU速度和数量无要求

  • 解决竞争条件：

    • 软件实现 — （turn； flag）—不满足前进性； flag+turn-Peterson算法
      • turn — do {while(turn != i); Enter; trun=j;} — 别人来给自己
      • flag 和 peterson — 自己给自己权限 — 在 do while 内修改
    • 硬件实现 — 关中断(CLI STI)；特殊指令(TSL 或者 SWAP)
      • 关中断 — (CLI STI)特权指令 — 用户进程不允许关中断
      • bool TSL(bool &target){bool rv=target; target=true; return rv;}
      • void SWAP(bool &a, bool &b){bool temp=a;a=b;b=temp;}
      • TSL 和 SWAP 都是 原子操作 都不满足有限等待
    • 这些解决都有一个问题： 忙等 — 这种类型的解决叫 自旋锁
      • 一个进程在临界区中，其他想进入临界区的进程在进入区陷入持续循环
      • 自旋锁 只对 多处理器系统有用； 没有上下文切换—没下CPU

第八章 死锁

  • 死锁：集合中的每个进程都在等待只有集合内其他进程导致的事件
  • 使用资源的动作： 请求、使用、释放
  • 死锁4个必要条件：
    • 互斥
    • hold and wait(保留并等待)
    • 无抢占
    • 循环等待
  • 死锁描述 —- 资源分配图
    • 无环路，then 无死锁
    • 有环路，可能死锁
  • 解决死锁：死锁预防、避免、检测与恢复、鸵鸟算法
    • 死锁预防 — 打破必要条件
      • 互斥 — 将非共享资源转换为共享资源—对设备进行假脱机
      • 保留并等待 — 请求时不拥有资源；执行前拥有所有资源
        • 结果： 资源利用率低 或者 饥饿
      • 无抢占 — 如果进程请求无法获得的资源，那么自己拥有的所有资源也要释放。
      • 循环等待 — 对资源类型进行总排序，进程按照枚举的递增顺序进行资源请求
    • 死锁避免 — 银行家算法(精确记忆) — 分配资源前进行，避免进入不安全状态
      • Max，Allocation，Available=Work，Need
      • 安全状态 — 没有死锁；不安全状态 — 可能死锁
      • used in THE —- $m\times n^2$ 动作
    • 死锁检测与恢复—精确记忆
      • Allocation、Request、Available
      • 死锁检测 — 类似银行家算法
      • 恢复 — 进程终止(所有死锁进程；一次终止一个进程直到没有死锁）；资源抢占
    • 鸵鸟算法 — UNIX/Linux;Windows

第九章 内存管理

  • 内存管理方法：覆盖和交换
    • 交换—进程换出到后备存储。换进时必须在相同的内存地址处
  • 内存管理问题： 重定位、内存管理、内存分配
  • 重定位： 编译时、加载时、执行时 绑定
  • 地址绑定：把程序中的符号映射为地址
  • 几个概念
    • 逻辑地址：CPU产生的地址
    • 物理地址：系统中内存单元看到的地址
    • MMU(memory management unit） 实现逻辑地址到物理地址转换的硬件
  • 内存保护
    • 对于CPU地址(像是物理地址) —- base、limit 寄存器
    • 对于逻辑地址 —- limit、重定位寄存器
  • 内存分配
    • 可用内存块 — hole(孔) — 分配，释放并邻近合并
    • 外部碎片：留下的hole不足以满足任何进程的需求，且不能和邻近的hole合并
    • 内部碎片：分配给进程的内存比要求的多
    • 分配算法： 连续物理地址空间分配—-首次适应，最佳适应，最坏适应
      • 首次适应、最佳适应—-外部碎片 最坏适应—内部碎片
    • 解决外部碎片—-紧缩(compaction) 或者 非连续物理地址空间分配(分页，分段，段页式)
    • 分页优点：
      • 非连续的物理内存分配
      • 没有外部碎片，有一定的内部碎片
      • 对进程的部分内存进行换入换出
      • 硬件和操作系统共同实现

  • PTE — page table entry

  • 由于是操作系统在控制地址映射，它必须

    ·记录系统物理内存的使用状况。一般使用一个称为frame table的数据结构来保S存系统中每一个frame的状态；

    ·为每一个进程保存一个page table

  • page table 必须在内存中 — PTBR、PTLR寄存器 记录页表位置信息

  • 无TLBs： 一次内存访问两次内存操作 —- page table 和 内存

  • 有TLBs(translation look-aside buffers): 内存操作— 命中-1次；miss-2次

    • 但是不管命不命中， 都有一个访问TLBs的时间
    • $EAT = (1+\epsilon)\times \alpha + (2+\epsilon) \times(1-\alpha)$ a：命中率

  • 分页内存保护：PTE含有 保护信息RWX(只读，读写，执行) valid/invalid

  • 一个frame/page 装不下 整个pagetable -> 页表结构：层次、哈希、反向 页表

  • 层次型页表—- 外部页表(页表目录directory) PDE

  • 帧管理 —- 空闲帧链表

  • 以 页面为单位 进行内存共享

  • 分段： 不固定大小的段， 每个段存储一种类型的数据 产生外部碎片

    • logical address = <段号， 段偏移>
    • 段表 — STBR，STLR寄存器保存段表位置信息
    • STE = base，limit，保护位(RWX) ，不像PTE是帧号

  • IA-32 段页式存储典范

    • STE 被称为 描述符
    • 先分段的方式获得地址，在对这个地址用分页的方式查找物理地址

第十章 虚拟内存

  • 虚拟内存提供了 巨大的，连续的逻辑内存
  • 只将程序执行需要的部分 放入内存中 — lazy swapper
  • 虚拟内存实现通过：
    • 请求分页，请求分段
  • valid -》 地址合法 且 相关页在内存中
  • 地址转换查看页表发现是invalid；CPU触发 缺页异常；(后面的都是 缺页异常服务例程) OS检查 内部表 查看是否是合法内存访问；非法就终止进程；合法就检查有无空闲帧；有就直接将页面从 交换空间(辅助存储器) 读入 内存；没有就进行页面置换；返回中断的指令处继续执行
  • 多道程序 -> 物理内存的过度分配
  • 页面置换：没有空闲帧时，选择一个 受害者帧，根据 dirty bit 选择是否将其写回交换空间； 修改页表—变为无效； 置换； 修改页表； restart
  • 有空闲帧，一次修改页表；没有空闲帧，两次修改页表
  • 页面置换算法：
    • FIFO-Belady 异常，optimal，LRU，二次机会（访问位）
  • 颠簸(thrashing)： 进程换页所用时间多于CPU执行时间。 频繁的换页
    • 物理帧数少于进程最低要求-》频繁的换页-》低CPU利用率-》os认为CPU空闲，增加多道程序程度-》换页更频繁了
    • 缓解颠簸： 降低多道程序程度
    • why thrashing： 所有进程的局部之和 > 总的内存大小
      • 局部模型： 局部是一个进程在一段时间内频繁使用的页面集合
  • 工作集模型 — 是局部模型的近似 避免了thrashing，同时保持高的多道程序程度
    • $\bigtriangleup$ 考虑t时刻以及以前 多少个页面
    • $WSS_i$ 进程i的工作集(频繁使用的页面)
    • OS为每个进程模拟工作集， 并为每个进程分配 工作集大小的 帧数

第十一章 文件系统

  • 文件—逻辑连续的辅助存储

  • 文件操作： 创建，打开，关闭，读，写，搜索，删除

  • 决定文件类型的是文件内容

  • 文件访问 — 顺序访问，随机访问

  • 文件系统驻留在设备上，设备被拆分为文件系统所在的一个或多个分区

  • 目录结构：单级目录，二级目录(用户),树结构，无环图(相同文件不同名字，eg桌面图标)

  • 文件系统挂载（mounted）

    ​ ·驻留在设备上的文件系统必须挂载后才能访问。

    ·挂载文件系统的位置称为挂载点。

    ·通常，挂载点是一个空目录。

  • UID，GID 三类用户 owner，group，public

  • ACL - access control list — 用户名 和 访问类型

  • FCB — file control block

  • 目录实现： 线性表，哈希表

  • 分配方法(磁盘数据块分给文件)：

    • 连续分配 — 开始块和长度in FCB；随机访问；外部碎片
    • 链接分配 — first last pointer in FCB；next pointer in Block；无随机访问
      • FAT MS-DOS — 分区放file Allocation table 含所有指针
    • 索引分配 — 所有指针在某个数据块里面
      • 随机访问
      • 无外部碎片

  • 空闲空间管理—位向量 链表
